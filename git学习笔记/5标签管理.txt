tag 标签管理
可以对某一次的commit进行一个标识，相当于起了一个别名。

比如一个版本，针对最后一次commit起一个v1.0这样的标签标识里程碑的意义。

比如还有比较难以记住的commit id，tag可以很好的解决。reset版本回退可以很快捷的定义标签的修改版本。

----------------新增标签
命令：git tag v1.0
	此时会给最新的一次提交打上标签
git tag 可以查看当前的所有标签
在.git文件下 refs下的tags下存在对应的标签名文件，而此文件实际上内部就是存的最新一次提交的commit-id罢了。


命令：git tag v0.9 commit-id
	会给对应commit-id打上对应的标签。
打标签的列表是根据英文单词排序的，不是按照时间进行的。


--------------标签添加描述
命令 git tag -a 标签名 -m "描述"
	可以给对应的标签加上描述。


------------查看标签详细信息
命令：git show 标签名
	可以看到对应标签名的一个详细信息以及对应的描述。


----------------删除标签
命令：git tag -d 标签名
	删除对应标签



----------------------推送标签
上述操作只是在本地进行了操作，并没有推送到远程仓库上。

git push origin 标签名
	直接可以推送对应标签即可。


一次性推送所有标签
git push origin --tags


在本地删除标签，并且同步到远端。
git tag -d 标签名
git push origin :标签名 
	此时可以将对应远端仓库中的标签名给删除掉。
